## Decisions Made
- Obsidian structure: project > ticket/topic > dated notes
- Top folder = project name (repo folder name, e.g. `silia`)
- `Active Context.md` = single recovery entry point, always overwritten
- Session notes = detailed play-by-play, only appended when something changes
- Auto-save every 10 min but SKIP append if nothing changed (avoid spam)
- Notes must be DETAILED, never summarized
- CLAUDE.md prints `[obsidian]` logs on startup so user sees each step

## Problems / Errors Encountered

- `obsidian write` doesn't exist — use `obsidian create ... overwrite`
- Auto-save appended 35+ empty "no changes" checkpoints — fixed by making append conditional
