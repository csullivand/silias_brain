# Knowledge Vault

## Purpose
This vault is my personal knowledge base. All relevant information should be saved here as Markdown notes.

## Note Conventions
- Use wikilinks to connect related concepts
- One idea per note (atomic notes)
- Use aliases in frontmatter when a note has multiple names
- Use block references (^id) to cite specific sections
- File names in Title Case with spaces: My Important Note.md
- YAML frontmatter with tags and creation date

## Folder Structure
- /projects — notes per active project
- /concepts — definitions and explanations
- /references — information from external sources
- /journal — daily work notes
- /Templates — reusable templates
- /Claude Sessions/{project}/{topic}/{date}.md — work sessions with Claude

## Rules
- ALWAYS check existing notes before creating duplicates
- Connect new notes to existing ones using wikilinks

## Claude Work Sessions
- When creating a new session, use the [[Session Template]]
- When finishing a session, save a detailed summary in the corresponding session note: Claude Sessions/{project}/{topic}/{date}.md
- Update [[Active Context]] with the latest checkpoint: what was done, current state, and how to continue
- Update [[Claude Sessions/Index]] if a new session was created or a topic's status changed
