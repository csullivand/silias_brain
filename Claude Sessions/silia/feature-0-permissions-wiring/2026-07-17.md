Feature 0 tiene 3 fases: **Fase 1** = catálogo de 102 permisos atómicos (`scripts/migrations/permissions-catalog.json`, con `name`+`label`+`category`, seed `seed-permissions-catalog.ts`). **Fase 2** = matriz rol→permiso (`role-permissions.json`, 102 asignaciones transcritas del PRD, seed `seed-role-permissions.ts`). **Fase 3 (esta sesión)** = cablear cada endpoint HTTP para que exija su permiso atómico vía `assertPermission(event,'x')` / `assertObjectPermission(event,'x',type,id)` (llamada en código) o `CASL_PERMISSION` env-var (patrón Middy `requirePermission()`, usado por Billing). Corre en **modo permisivo** (`PERMISSION_STRICT_MODE=off`): si falta el permiso **advierte** pero no bloquea; se preserva el `assertRole` previo como piso (cero regresión). El flip a strict mode es Fase 6.
