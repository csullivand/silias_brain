### 4. Backend Enforcement (JWT-based)
- **Approach:** accountStatus in JWT → authorizer passes to context → middleware reads it (zero DB calls per request)
- **Auth/login.ts:** Added accountStatus to JWT payload via generateTokens(user, account.status)
- **Auth/middleware.ts:** Passes decoded.accountStatus through authorizer context
- **shared/middleware/requireActiveAccount.ts:** New Middy middleware
  - Reads accountStatus from authorizer context
  - Returns 403 with code ACCOUNT_SUSPENDED if not active
  - Superusers bypass
  - Short-circuits via request.response (not throw) for correct middy behavior
  - Also provides assertActiveAccount() utility for callback-based handlers
- **Token refresh gap:** refresh.ts doesn't include accountStatus (pre-existing issue, not fixed)
  - Refreshed tokens default to 'active' in middleware
  - Gap is max session duration (~24h) vs 7-day dunning — negligible risk
  - All chatbots already status=0 when suspended, agents don't respond regardless
