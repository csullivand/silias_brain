## Decisions
- Used composite key (userId+tableId) instead of single PK + GSI — every query uses both
- PUT validates column IDs exist before saving (prevents stale references at write time)
- GET filters deleted columns at read time (handles columns deleted after config saved)
- No separate module — this belongs in DynamicTables
