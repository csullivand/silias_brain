# DynamicTables Refactor — rowCount Feature + Deploy Fixes

## Context
Continuing work on PR #1353 for DynamicTables refactor. This session focused on fixing Adversarial Verify BLOCK issues for the rowCount feature.

## Work Done

### Adversarial Verify BLOCK Fixes
Two BLOCK issues were identified by the adversarial verify shadow process:

1. **correctness BLOCK** — `DYNAMIC_TABLES_ROWS_TABLE` env var missing from Folders Globals
   - **Problem:** `DynamicRowModel.getActiveRowCount()` uses `process.env.DYNAMIC_TABLES_ROWS_TABLE` but Folders Lambda didn't have this env var
   - **Impact:** rowCount would always return 0 (empty string table name → no results)
   - **Fix:** Added `DYNAMIC_TABLES_ROWS_TABLE: !Sub ${StackName}-DynamicTables-Rows` to Folders/infrastructure/aws.template.yml Globals

2. **infra BLOCK** — Deploy order not documented
   - **Problem:** Folders exports `Folders-KmsKeyArn`, Assistant and DynamicTables import it. If Folders deployed after, CFN fails
   - **Fix:** Added deploy order comments to all 3 templates:
     - Folders: "must be deployed BEFORE Assistant and DynamicTables"
     - Assistant: "must be deployed AFTER the Folders stack"
     - DynamicTables: "must be deployed AFTER the Folders stack"
